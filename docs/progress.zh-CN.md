# 项目进度时间线

本文档是项目变更记录的中文版本。每次代码或文档更新后，都要在本文档和 `docs/progress.md` 中各追加一条记录。记录应按时间从旧到新排列，最新记录放在末尾。内容重点说明改了什么、实现或改善了什么功能，以及未来可以优化什么，避免展开过多技术实现细节。

## 2026-06-27

### 变更

- 新增项目规则：后续每次变更都要记录到进度时间线。
- 创建英文进度文档，作为项目历史记录的固定位置。

### 功能结果

- 项目进展有了统一、可追溯的记录位置。
- 后续成员可以按时间顺序查看已完成的工作。

### 未来优化

- 随着项目扩大，可以按里程碑或版本整理记录，同时保留时间线顺序。

## 2026-06-27 — Milestone 0 项目脚手架

### 变更

- 建立了前后端、共享包、配置、基础文档和本地开发框架。
- 准备了数据库模型、容器化开发环境、基础健康检查和持续集成骨架。

### 功能结果

- 项目从空目录变成可安装、可构建、可继续开发的应用基础。
- 后续功能可以在统一的工作区、命令和目录结构中推进。

### 未来优化

- 当依赖下载环境更稳定后，可以移除临时的 Next.js SWC 启动辅助方案。

## 2026-06-27 — Milestone 1 基础设施、数据模型和种子数据

### 变更

- 启动并验证了本地 PostgreSQL 和 MinIO 私有对象存储。
- 应用了初始数据库迁移，补充了可重复执行的种子数据。
- 增加了数据库和对象存储健康检查。

### 功能结果

- 本地环境可以提供真实数据库和私有文件存储。
- 开发者可以使用固定测试账号、课程、邀请、资料和笔记数据体验系统。
- 私有存储不会向匿名用户公开资料文件。

### 未来优化

- 后续可以增加更接近生产环境的恢复演练和对象清理策略。

## 2026-06-27 — Milestone 2 认证和全局授权

### 变更

- 实现邀请注册链接、登录、登出、当前用户查询和账号禁用流程。
- 引入服务器端会话、会话撤销、同源保护和 CSRF 防护。
- 为受保护页面和接口增加默认鉴权。

### 功能结果

- 未登录用户只能访问登录、注册和健康检查相关入口。
- 学生必须通过有效邀请完成注册后才能进入系统。
- 管理员可以禁用账号并撤销相关会话。

### 未来优化

- 可以继续增强登录和邀请相关的限流、告警和审计展示。

## 2026-06-27 — Milestone 3 学期、课程、成员和邀请

### 变更

- 增加学期、课程、课程成员和邀请管理能力。
- 课程管理员可以在自己的课程范围内管理成员和邀请。
- 前端接入了仪表盘、课程详情、资料入口、个人资料和管理入口。

### 功能结果

- 用户可以按课程组织学习资料和成员权限。
- 系统管理员可以管理全站课程，课程管理员可以处理课程内事务。
- 邀请码注册会正确关联课程成员身份。

### 未来优化

- 可以增加用户搜索和批量成员管理，减少人工邀请操作。

## 2026-06-27 — Milestone 4 资料上传、版本和下载

### 变更

- 增加资料上传、资料详情、元数据编辑、新版本上传、软删除和恢复。
- 增加文件大小、类型、配额和重复文件检查。
- 下载前会进行授权，并返回短时有效的私有下载链接。

### 功能结果

- 授权学生可以安全上传和下载课程资料。
- 资料更新会保留历史版本，不会直接覆盖旧文件。
- 只读成员可以浏览和下载，但不能上传或修改内容。

### 未来优化

- 批量下载仍可在单文件权限、审计和存储流程稳定后再加入。

## 2026-06-27 — Milestone 5 搜索、浏览和预览

### 变更

- 增加资料元数据搜索、分类和标签筛选、排序和分页。
- 增加 PDF、图片、Markdown、文本、源码和笔记本文件的安全预览状态。
- 对 Office 和 ZIP 等不适合预览的文件保留下载型展示。

### 功能结果

- 用户可以更快按标题、课程、分类、标签、上传者和文件名查找资料。
- 常见学习资料可以在浏览器内安全查看。
- 上传内容不会被执行，预览过程更符合私有学习平台的安全边界。

### 未来优化

- 可以在未来引入全文内容搜索，但 MVP 阶段仍保持元数据搜索。

## 2026-06-29 — Milestone 6 笔记、修订、收藏和个人资料

### 变更

- 增加 Markdown 笔记的草稿、发布、编辑、删除、恢复和修订恢复。
- 增加资料和笔记收藏，以及个人资料页的上传、笔记、收藏和活动摘要。
- 前端接入笔记列表、新建/编辑页面、详情页、收藏页、课程空间和个人资料页。

### 功能结果

- 用户可以创建和分享课程笔记，并保留可恢复的历史版本。
- 用户可以收藏常用资料和笔记，个人中心能展示自己的学习内容活动。
- 笔记渲染支持常用 Markdown 能力，同时保持安全过滤。

### 未来优化

- 可以增加更友好的修订对比视图，帮助用户查看版本差异。

## 2026-06-29 — Milestone 7 管理界面和审计

### 变更

- 增加用户管理、资源管理和审计日志页面。
- 管理员可以搜索用户、禁用/启用用户、撤销会话、查看和处理资源。
- 审计日志支持按动作、课程、操作者和目标过滤，并按权限限制可见范围。

### 功能结果

- 管理员可以在界面中处理账号、资源和安全审计事务。
- 课程管理员只能查看和处理自己课程范围内的内容。
- 系统对高风险管理操作保留可追溯记录。

### 未来优化

- 可以增加批量用户导入和审计日志导出。

## 2026-06-29 — Milestone 8 可靠性、CI、备份和部署

### 变更

- 增加生产构建镜像、就绪检查、CI 流程和更完整的端到端测试覆盖。
- 增加数据库和对象存储的备份、恢复脚本与本地恢复演练。
- 更新部署、备份和恢复文档。

### 功能结果

- 项目可以通过自动化流程验证格式、类型、测试、构建和浏览器场景。
- 运维人员可以按文档备份和恢复数据库及资料对象。
- 部署前可以通过就绪检查确认核心依赖可用。

### 未来优化

- 可以增加定时备份、异地复制和长期对象清理任务。

### 变更

- 完成最终仓库审查，清理生成物和调试残留。
- 补充桌面和移动视口的主页面冒烟测试。

## 2026-06-30 — CI MinIO 启动修复

### 变更

- 将 GitHub Actions 中的 MinIO 启动方式改为显式 Docker 步骤，并传入必需的 `server /data` 命令。
- 从 GitHub Actions service container 列表中移除 MinIO，同时保留 PostgreSQL service container。
- 继续在确认 MinIO 就绪后执行私有 bucket 初始化。

### 功能结果

- CI 不再因为 MinIO 缺少启动命令而在 GitHub Actions 容器初始化阶段失败。
- 数据库迁移、种子数据、验证命令和浏览器测试仍可使用同一个本地私有 S3 兼容端点。

### 未来优化

- 如果 GitHub Actions 后续支持为 service container 直接配置启动命令，可以再把 MinIO 移回 service container 配置。
