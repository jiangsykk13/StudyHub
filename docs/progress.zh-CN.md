# 项目进度时间线

本文档是项目变更记录的中文版本。每次代码或文档更新后，都要在本文档和 `docs/progress.md` 中各追加一条记录。记录应按时间从旧到新排列，最新记录放在末尾。内容重点说明改了什么、实现或改善了什么功能，以及未来可以优化什么，避免展开过多技术实现细节。

## 2026-06-27 — Milestone 1 基础设施、数据模型和种子数据

- 引入服务器端会话、会话撤销、同源保护和 CSRF 防护。

## 2026-06-27 — Milestone 4 资料上传、版本和下载

## 2026-06-30 — CI MinIO 启动修复

### 变更

- 将 GitHub Actions 中的 MinIO 启动方式改为显式 Docker 步骤，并传入必需的 `server /data` 命令。
- 从 GitHub Actions service container 列表中移除 MinIO，同时保留 PostgreSQL service container。

### 功能结果

## 2026-06-30 — CI 原生依赖安装修复

## 2026-06-30 — CI 安装步骤稳定化

### 未来优化

### 未来优化

## 2026-06-30 — CI AJV peer 解析修复

### 变更

- 新增显式根目录 `ajv` 开发依赖，在项目仍关闭大范围 optional 安装的情况下，让 `ajv-formats` 的 optional peer 链接可以解析。

- CI 仍会检查关闭 optional 安装后可能导致验证失败的 Rollup、Turborepo、esbuild、lightningcss 和 Tailwind Oxide 原生包。

### 未来优化

- 如果后续包管理器版本可以同时稳定处理 optional peer 和平台原生 optional 依赖，可以再评估是否保留显式 AJV peer 和原生工具引导脚本。

## 2026-06-30 — CI E2E 启动稳定化

### 变更

- 新增只在 CI 使用的 Playwright webServer 启动脚本，直接启动已构建的 Nest API 和 Next.js 生产服务器，不再使用较慢的根目录开发命令。
- 本地 Playwright 运行继续使用现有开发启动路径，并适当提高 CI webServer 超时时间。

### 功能结果

- GitHub Actions E2E 可以复用 `pnpm verify` 已生成的构建产物，并在浏览器测试开始前等待真实 API 和 Web 健康检查通过。

### 未来优化

- 如果后续 E2E 套件需要每次运行隔离测试数据库，可以扩展该启动脚本，在启动服务前为临时数据库执行迁移和种子数据。
